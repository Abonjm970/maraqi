# موقع برنامج "مراقي المعرفة" القرائي

> نسخة محدّثة — تتضمن حلول الفجوات المكتشفة أثناء المراجعة، بناءً على ثلاث قرارات:
> زر "قراءة" يفتح PDF من Google Drive، كل حقيبة فيها 3 كتب بالضبط، والاستضافة على GitHub Pages.

## التعريف بالموقع
موقع "مراقي المعرفة" هو موقع لبرنامج قرائي بسيط.

## الفئة المستهدفة
طلبة علم ناشئون بعمر بين 12-19 سنة، قد لا يكون لديهم خبرة كافية في استخدام مواقع الإنترنت.

## الاستضافة
الموقع سينشر على **GitHub Pages**. بما إن هذا قد يعني نشره كمشروع فرعي (مثل `username.github.io/repo-name`)، فكل الروابط الداخلية (CSS، الشعار، الروابط بين الصفحات) تُكتب بمسارات **نسبية** (`../style.css`)، لا مسارات من الجذر (`/style.css`) — عشان تشتغل صح بأي طريقة نشر يختارها المستخدم لاحقًا.

# تصميم الموقع

## السمة العامة
- برنامج للقراءة والكتب
- الألوان (كمتغيرات CSS في style.css؛ القيم بداية قابلة للتعديل):
  - `--color-white: #FFFFFF`
  - `--color-gold: #C9A227`
- كل الصفحات تحتوي شعار البرنامج في الأعلى، والضغط عليه يؤدي إلى الصفحة الرئيسية
- زخرفة إسلامية في الخلفية
- `dir="rtl" lang="ar"` إلزامي في وسم `<html>` بكل صفحة بدون استثناء
- التصميم متجاوب (mobile-first) — الفئة المستهدفة غالبًا تتصفح من الجوال
- ملف `style.css` فيه متغيرات الألوان والخطوط، يُستخدم في كل الصفحات

## الهيدر المشترك
الشعار وشريط التنقل مكرران في 9 صفحات (الرئيسية + مكتبتان + 6 حقائب + صفحة الكتاب). بدل تكرارهما يدويًا في كل ملف:
- `header.html` ملف جزئي (partial) يُحمَّل ويُحقن بجافاسكربت وقت تحميل كل صفحة (`fetch('header.html')`)
- يشتغل مباشرة على GitHub Pages بدون أي أدوات بناء إضافية (build tools)
- أي تعديل مستقبلي على الشعار/التنقل يصير في ملف واحد فقط

# الصفحات

## الصفحة الرئيسية (index.html)
تحتوي على:
- اقتباس:
  > «فمن طلبَ العلمَ ليحيي به الإسلامَ فهو من الصِّدِّيقين، ودرجتُه بعد درجة النبوَّة.»
  > `- [مفتاح دار السعادة 1/ 339]`
- بطاقتان (تم تصحيح تكرار النص في الخطة الأصلية):
  - **مكتبة المرحلة المتوسطة** — دخول (زر) ← `mid/mid.html`
  - **مكتبة المرحلة الثانوية** — دخول (زر) ← `high/high.html`

وقد يُضاف عليها في المستقبل أشياء أخرى.

## مكتبة المتوسط (mid/mid.html)
3 بطاقات (حقائب):
- الحقيبة الأولى (تراجم - سلوك) — فتح ← `bag_1/bag_1.html`
- الحقيبة الثانية (إيماني - أدبي) — فتح ← `bag_2/bag_2.html`
- الحقيبة الثالثة (مهاري - سيرة) — فتح ← `bag_3/bag_3.html`

## مكتبة الثانوي (high/high.html)
3 بطاقات (حقائب):
- الحقيبة الأولى (سلوكي - إيماني - تراجم) — فتح ← `bag_1/bag_1.html`
- الحقيبة الثانية (علمي - مهاري - تراجم) — فتح ← `bag_2/bag_2.html`
- الحقيبة الثالثة (سيرة - إيماني - علمي) — فتح ← `bag_3/bag_3.html`

## صفحات الحقائب (bag_X.html)
كل حقيبة تحتوي **3 كتب بالضبط** (ثابت حسب القرار). الصفحة تقرأ `books.json` المجاور لها وتبني البطاقات بحلقة (loop) بدل كتابتها يدويًا — أسهل صيانة لو تغيّر كتاب لاحقًا.

كل بطاقة كتاب تعرض:
- صورة الكتاب (من books.json)
- اسم الكتاب
- تصنيف الكتاب
- زر "قراءة"

### مخطط books.json
```json
{
  "id": "book1",
  "title": "اسم الكتاب",
  "category": "تصنيف الكتاب",
  "cover": "book1.png",
  "drive_url": "رابط لملف pdf على google drive"
}
```

# ملاحظات تطوير
- شغّل الموقع أثناء التطوير عبر سيرفر محلي — `python -m http.server` أو إضافة Live Server بـ VS Code. فتح الملفات مباشرة (`file://`) يكسر `fetch()` الخاص بـ books.json وheader.html.
- كل الروابط الداخلية نسبية دائمًا، بسبب GitHub Pages.
- بعد أي تعديل بصري، تحقّق بـ chrome-devtools MCP: افتح الصفحة، خذ سكرين شوت، تأكد من الخط والاتجاه والألوان قبل الانتقال لصفحة أخرى.
