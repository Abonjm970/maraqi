# مراقي المعرفة — تعليمات المشروع لـ opencode

موقع ثابت (static HTML/CSS/JS، بدون إطار عمل أو backend) لبرنامج قرائي إسلامي، بالعربية بالكامل، لفئة عمرية 12-19 سنة قليلة الخبرة بالإنترنت. سينشر على GitHub Pages.

## قواعد إلزامية على كل صفحة
- `<html dir="rtl" lang="ar">` في كل ملف HTML بدون استثناء.
- الهيدر (الشعار + رابط الرئيسية) يُحقن من `header.html` عبر `fetch` وقت التحميل — لا يُكتب يدويًا بكل صفحة.
- كل الروابط والمسارات (CSS، الشعار، الصفحات) نسبية (`../`)، أبدًا من الجذر (`/...`) — بسبب احتمال نشر GitHub Pages بمسار فرعي.
- التصميم متجاوب (mobile-first).

## الخطوط والألوان
معرّفة كمتغيرات CSS في `style.css`، لا تُكتب كقيم متفرقة بالملفات:
- `--color-white: #FFFFFF`
- `--color-gold: #C9A227`
- ممنوع استخدام أي خط لا يدعم العربية للعناوين أو المتن.

## books.json — مصدر الحقيقة الوحيد
- كل حقيبة (`bag_X/`) فيها `books.json` يحتوي 3 كتب بالضبط.
- بطاقات الكتب تُبنى بحلقة (loop) تقرأ من `books.json` — لا تُكتب 3 بطاقات يدويًا بالـ HTML حتى لو العدد ثابت.
- كل عنصر: `id`, `title`, `category`, `cover`, `drive_url`
- زر "قراءة" يفتح رابط `drive_url`

## أثناء التطوير
- لازم سيرفر محلي (`python -m http.server` مثلًا) — فتح الملفات مباشرة (`file://`) يكسر `fetch()` لـ `books.json` و`header.html`.
- بعد أي تعديل بصري، تحقّق بـ chrome-devtools MCP: افتح الصفحة، خذ سكرين شوت، تأكد من الخط والاتجاه (RTL) والألوان قبل الانتقال للصفحة التالية.

## نمط الصفحات المتكررة
`index.html` ← `mid/mid.html` أو `high/high.html` ← `bag_X/bag_X.html`

كل `bag_X.html` بنفس البنية بالضبط، فقط المحتوى (من `books.json`) يتغيّر.
